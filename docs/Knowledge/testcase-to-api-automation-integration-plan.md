# 测试用例生成连接 API 自动化可行性方案

> 更新时间：2026-05-08

## 可行性结论

可行。推荐先实现“一键转 API 自动化脚本”的低风险链路：测试用例生成完成后，将已解析的用例和关联 API 规范转换为 API 自动化 DSL，跳转到 API 自动化编排页，由用户确认环境、策略和脚本后再执行。

不建议一期直接做“生成后自动执行”，避免绕过 API 自动化已有的环境配置、策略审计和人工确认边界。

## 一期范围

- OpenAPI 文件优先打通：复用现有 API 自动化的 OpenAPI 解析和 DSL 生成能力。
- 文本需求也支持接入，但必须先关联已有 API 项目或接口文档，不能无依据地从自然语言猜测接口。
- 未匹配到接口的测试用例保留为人工用例，不强行转换为 API 步骤。
- 转换结果只生成 DSL，不直接创建执行记录，也不自动发起运行。

## 推荐工作流

1. 用户在“测试用例生成”页面上传 OpenAPI 文件或输入文本需求。
2. 系统生成 Markdown 测试用例，并解析为结构化 test cases。
3. 结果操作区提供“转为 API 自动化脚本”入口。
4. 用户点击后，系统基于 `spec_id` 和测试用例匹配接口 operation。
5. 后端返回转换结果，包括生成的 DSL、已匹配接口、未匹配用例和提示信息。
6. 用户确认后跳转到 API 自动化页面，自动载入 DSL，并停留在编排执行步骤。
7. 用户在 API 自动化页面确认环境、策略、Header、变量和脚本后手动执行。

## UI 计划

测试用例生成结果区新增按钮：

- `转为 API 自动化脚本`

按钮启用条件：

- OpenAPI 文件模式：已完成文件解析并能获得 `spec_id`。
- 文本需求模式：已选择关联 API 项目或接口文档。
- 当前已解析出至少一条结构化测试用例。

转换后展示：

- 匹配到的用例数。
- 生成的 API 步骤数。
- 未匹配用例列表。
- 低置信度匹配提示。

确认后跳转 API 自动化页面，并自动填充 `dslText`。

## 后端接口计划

新增接口：

```http
POST /api/api-execution/dsl/from-test-cases
```

请求体：

```json
{
  "spec_id": "string",
  "test_cases": [],
  "match_mode": "safe"
}
```

响应体：

```json
{
  "script": {},
  "matched_operations": [],
  "unmatched_cases": [],
  "warnings": []
}
```

接口行为：

- 根据 `spec_id` 获取已解析 API 规范。
- 根据测试用例内容匹配 operation。
- 复用现有 `generate_api_dsl()` 生成标准 `APITestCaseDsl`。
- 低置信度和无匹配用例进入 `unmatched_cases`。
- 不执行脚本，不写执行历史。

## 匹配策略

匹配优先级固定为：

1. `operation_id`
2. `METHOD path`
3. path 片段
4. summary / title 关键词

生成提示词可增强输出字段：

```markdown
**关联接口:** GET /api/users (listUsers)
```

该字段作为最高优先级匹配线索。若缺失该字段，再使用标题、描述、步骤和预期结果进行辅助匹配。

## 安全边界

- 只生成 DSL，不自动执行。
- 不自动选择生产环境。
- 不自动应用 AI 修复或重跑。
- 高风险接口仍交给 API 自动化现有策略判断。
- 未匹配用例不强制转换。
- 用户必须在 API 自动化页面确认后才能运行。

## 测试计划

后端测试：

- OpenAPI spec + 带 `关联接口` 的测试用例能生成 `APITestCaseDsl`。
- 仅有标题或步骤关键词时能匹配 summary/path。
- 无匹配时返回 `unmatched_cases`，不生成错误步骤。
- `spec_id` 不存在时返回明确错误。

前端测试：

- OpenAPI 文件生成后可以触发转换并跳转 API 自动化页。
- 文本模式未关联 API 项目时禁用转换入口或给出提示。
- 存在未匹配用例时展示确认信息。
- 原有生成、导出、存向量流程不受影响。

回归测试：

- API 自动化现有 DSL 生成、校验、执行、导出流程保持不变。
- 测试用例生成结果解析逻辑保持兼容。

## 默认假设

- 一期只做“一键转 API 自动化脚本”，不做“生成后自动执行”。
- 文本需求必须关联已有 API spec 才能转换。
- 转换结果只影响前端状态，不直接写执行历史。
- 未匹配用例保留为人工测试用例。
- 文档新增不更新 `CHANGELOG.md`。
