# OpenMelon 演示指南

本文档将一步步展示系统的核心功能，帮助你在 10 分钟内快速上手。

---

## 使用前提（只讲操作，不讲部署）

本指南只包含系统内的页面操作流程，不包含安装、启动、部署步骤。
如果你还没把系统启动起来，先看 [README.md](README.md) 的快速开始，或看 [MANUAL.md](MANUAL.md) 的安装与启动章节。

默认你已经有可访问的系统地址，并且可以正常登录/打开页面。建议按下面顺序演示：

1. 导入管理：上传文件并看到“已索引”
2. 问答：提一个问题并看到带引用的回答
3. 图谱总览：搜索实体并看到关系子图
4. 测试用例生成：生成并导出用例
5. 覆盖率视图：展示模块风险和覆盖率排行

### 开始前检查（页面层）

| 检查项 | 正常表现 |
|------|------|
| 首页可访问 | 能进入系统并看到顶部导航 |
| 导入管理 | 上传区域可选文件 |
| 问答页 | 输入框可以发送消息 |
| 图谱总览 | 图谱区域可正常展示 |
| 测试用例生成 | 能切换“文件生成/文本描述”模式 |

## 场景一：上传文档并索引

> **目标**：将文档上传到系统，自动解析、分块、构建知识图谱和向量索引。

### 操作步骤

1. **进入「导入管理」页面**
   点击顶部导航栏的「导入管理」Tab。

2. **选择上传模式**
   左侧导入工作台默认为「单文件上传」，可切换为「文件夹批量上传」。

3. **选择文件**
   - 将文件拖拽到上传区域，或
   - 点击上传区域选择文件

   支持 16 种格式：PDF / Word / Excel / XMind / PPT / Markdown / TXT / CSV / JSON / YAML / HTML / EPUB 等。

4. **填写可选信息**
   - **文档类型**：例如「需求文档」「接口文档」「测试报告」
   - **模块名**：例如「用户管理」「订单系统」「数据审核」
   - 这两项会写入知识图谱，帮助后续按类型和模块筛选

5. **点击「开始导入」**
   系统立即保存文件，然后在后台异步处理。你可以继续使用系统，不需要等待。

6. **查看处理进度**
   上传区下方显示进度条，后台正在执行：
   -  文件解析（提取纯文本）
   -  文档分块（800 字符/块，100 字符重叠）
   -  Embedding 生成（并发调用 LLM API）
   -  向量索引写入
   -  实体提取 + 知识图谱构建

7. **确认结果**
   右侧索引清单会刷新，新文件出现在列表中，状态显示「已索引」。顶部统计卡片自动更新文件数和分块数。

### 通过 API 上传

```bash
# 上传文件
curl -X POST "http://localhost:8000/api/upload/async" \
  -F "files=@需求文档.pdf" \
  -F "doc_type=需求文档" \
  -F "module=用户管理"

# 返回 task_id，用于查询进度
# {"task_id": "abc123", "status": "pending"}

# 查询处理进度
curl "http://localhost:8000/api/upload/status/abc123"
# {"task_id": "abc123", "status": "completed", "chunks": 7}
```

## 场景二：智能问答

> **目标**：基于已索引的文档和知识图谱进行问答，获取带引用的智能回答。

### 前提

已至少索引一个文档（完成场景一）。

### 操作步骤

1. **切换到「问答」页面**
   点击顶部导航栏的「问答」Tab。

2. **在底部输入框输入问题**
   例如：
   ```
   这个系统有哪些核心功能模块？
   ```

3. **发送问题**
   按 Enter 键或点击「发送」按钮。

4. **等待回答**
   系统会显示思考动画，依次执行：
   -  意图识别（判断问题类型）
   -  实体提取（从问题中抓关键词）
   -  多通道检索（自动选择图谱/向量/混合检索）
   -  LLM 生成回答

5. **查看回答**
   回答显示在聊天区域，底部标注引用来源：
   - `Vector: 需求文档.pdf` — 来自向量检索
   - `Graph` — 来自图谱查询
   - `Method: hybrid` — 使用了混合检索

### 不同类型的问题

| 你想问 | 示例问题 | 系统会怎么做 |
|-------|---------|------------|
| 查实体关系 | "登录模块包含哪些功能？" | 在知识图谱中查找 Module → Feature 关系 |
| 查概念方法 | "如何实现用户注册功能？" | 在向量索引中搜索最相似的文档片段 |
| 查影响范围 | "修改订单状态会影响哪些模块？" | 同时查图谱和向量，综合回答 |
| 画结构图 | "画出用户管理的结构图" | 返回图谱子图数据 + 可视化摘要 |

### Agentic RAG 多步推理（适合复杂问题）

当标准问答不够深入时，可以通过 API 使用 Agentic 模式：

```bash
curl -X POST "http://localhost:8000/api/query?use_agentic=true" \
  -H "Content-Type: application/json" \
  -d '{"question": "修改数据审核流程会对哪些模块产生影响？请详细分析"}'
```

系统会自动执行多轮迭代：
```
Step 1: 初始检索 → Retrieved 5 chunks → sufficiency score=0.45（不够充分）
Step 2: 改写查询 → 重新检索 → Retrieved 5 chunks → score=0.72（充分了）
→ 基于所有上下文生成最终回答
```

| 对比 | 标准问答 | Agentic RAG |
|------|---------|-------------|
| 检索次数 | 1 次 | 最多 3 次 |
| 查询优化 | 无 | LLM 自动改写查询 |
| 推理过程 | 不可见 | 返回每步的检索和评分 |
| 适用场景 | 简单问题 | 复杂分析问题 |

## 场景三：探索知识图谱

> **目标**：可视化查看已构建的知识图谱，直观了解文档结构和实体关系。

### 操作步骤

1. **切换到「图谱总览」页面**
   点击顶部导航栏的「图谱总览」Tab。

2. **查看全图**
   页面自动加载所有节点和关系，等待加载动画消失后图谱自动居中。

3. **搜索特定实体**
   在搜索框输入实体名称（如「数据审核」「登录」），按 Enter 搜索。图谱会聚焦到该实体，显示其 2 度关系网络。

4. **使用筛选器**
   - **文档类型下拉框**：只看特定类型文档的实体（如只看需求文档）
   - **模块下拉框**：只看特定模块的实体
   - **显示分块复选框**：勾选后显示 DocumentChunk 节点（默认隐藏）

5. **交互操作**
   -  拖拽画布移动视图
   -  鼠标滚轮缩放
   -  点击节点 → 高亮关联节点 + 显示详情面板
   -  点击空白区域 → 关闭详情面板

6. **节点详情面板**
   点击节点后，右侧弹出玻璃态悬浮面板，显示节点类型和所有属性。面板标题栏有箭头按钮，可收起/展开。

### 节点颜色说明

| 颜色 | 节点类型 | 含义 |
|------|---------|------|
| 🔵 蓝色 | Product | 产品节点 |
| 🟢 绿色 | Module | 模块节点 |
| 🟡 黄色 | Feature | 功能节点 |
| 🔴 红色 | API | 接口节点 |
| 🟣 紫色 | TestCase | 测试用例 |
| 🟠 橙色 | Defect | 缺陷节点 |
| ⚪ 灰色 | Entity | 通用实体 |
| 🩵 青色 | DocumentChunk | 文档分块 |

## 场景四：生成测试用例

> **目标**：基于文档和知识图谱，让 AI 自动生成结构化的测试用例。

### 操作步骤

1. **切换到「测试用例生成」页面**
   点击顶部导航栏的「测试用例生成」Tab。

2. **选择生成模式**
   左侧面板顶部切换：
   - **文件生成**：上传图像 / PDF / OpenAPI 文档作为输入
   - **文本描述**：直接输入文字描述，无需文件

3. **上传文件**（文件模式）
   - 拖拽文件到上传区，或点击选择
   - 支持格式：`.png` `.jpg` `.jpeg` `.gif` `.bmp` `.webp` `.pdf` `.json` `.yaml` `.yml`
   - 图片/PDF 类文件会发送到 LLM 视觉模型进行理解

4. **填写上下文信息**
   - **上下文框**：描述产品背景和功能范围
     例如："电商系统的用户管理模块，包含登录、注册、密码重置、个人信息管理四个功能"
   - **需求框**：描述需要测试的具体需求
     例如："覆盖正常流程和异常场景，包括边界值测试和安全测试"

5. **选择模块**（可选）
   - 下拉框选择已有模块，或手动输入新模块名
   - 填入模块名后，系统会自动从 Neo4j 检索该模块的图谱知识（功能结构、已有用例、实体关系）

6. **点击「生成测试用例」**
   按钮变为加载状态，右侧面板开始展示生成过程。

7. **实时查看三阶段进度**
   右侧出现三个可展开的阶段卡片：
   - **阶段 1：需求分析** — AI 分析输入内容，提取结构化需求
   - **阶段 2：用例生成** — AI 基于分析结果生成完整测试用例
   - **阶段 3：用例评审** — AI 评审用例的完整性和可执行性

   当前正在生成的阶段自动展开，Markdown 内容实时流式输出。你可以自由滚动查看已生成的内容。

8. **查看结果**
   生成完成后，顶部显示统计卡片（用例总数、各优先级数量）。可以：
   - 切换「列表」和「导图」视图
   - 使用模块和优先级下拉框筛选

9. **存入向量库**
   点击「存入向量库」按钮，将生成的用例永久保存到 Neo4j 向量库。按钮旁边有 Tooltip 提示当前状态。
   - 向量库可用时：点击即可存储
   - 向量库不可用时：按钮置灰并提示原因

10. **导出用例导出**
    点击「用例导出」按钮，可以直接下载结构化的 Excel 表格。点击侧边的下拉箭头，还可以选择「导出为 XMind」，直接下载支持以主流原生软件打开的图文 `.xmind` 脑图文件。

### 通过 API 生成

```bash
# 从文件生成
curl -X POST "http://localhost:8000/api/test-cases/generate" \
  -F "file=@login_page.png" \
  -F "context=电商系统登录页面" \
  -F "requirements=覆盖正常和异常场景" \
  -F "module=用户管理"

# 从文本描述生成（无需文件）
curl -X POST "http://localhost:8000/api/test-cases/generate-from-context" \
  -F "context=用户管理系统，包含登录、注册、密码重置功能" \
  -F "requirements=覆盖正常流程和异常场景" \
  -F "module=用户管理"
```

两个接口都返回流式 Markdown 响应（`text/markdown`），按阶段标记分隔内容。

## 场景五：查看覆盖率

> **目标**：了解各模块的测试覆盖情况，识别薄弱模块。

### 操作步骤

1. **切换到「覆盖率视图」页面**

2. **查看指标卡片**
   页面顶部展示四个核心指标：
   - 模块总数
   - 功能总数
   - 用例总数
   - 风险模块数（覆盖率 < 50%）

3. **查看总览环图**
   展示系统的平均覆盖率。

4. **查看覆盖率排行**
   横向条形图，展示覆盖率最高/最低的模块，一眼识别薄弱环节。

5. **查看模块明细表**
   每行一个模块，显示：模块名、功能数、用例数、覆盖率进度条。

   进度条颜色含义：
   - 🟢 绿色（≥ 80%）：覆盖充分
   - 🟡 黄色（50-79%）：需要关注
   - 🔴 红色（< 50%）：存在风险

## 场景六：管理已索引文件

> **目标**：查看、维护和管理已导入系统的文件。

### 操作步骤

1. **切换到「导入管理」页面**

2. **查看统计信息**
   右侧顶部统计卡片显示：索引文件数、分块总数、覆盖模块数。

3. **筛选文件**
   - **日期筛选**：全部 / 今日 / 本周 / 本月

   - **搜索框**：按文件名模糊搜索

4. **执行操作**
   | 操作 | 说明 |
   |------|------|
   | 重新索引 | 从 `backend/app/data/uploads/` 读取原始文件重新处理 |
   | 删除 | 清理索引记录 |
   | 批量删除 | 全选/多选后一次性删除 |
   | 生成用例 | 从该文件直接发起测试用例生成 |

## 进阶功能

### 会话管理

问答页面左侧有会话侧边栏：
- 点击 `+` 新建会话
- 点击列表条目切换会话
- 点击条目右侧 `×` 删除会话

> 注意：会话数据存储在内存中，服务重启后丢失。

### 日志查看

```bash
# 查看运行日志（最近 100 行）
curl "http://localhost:8000/api/logs?filename=openmelon.log&lines=100"

# 查看错误日志
curl "http://localhost:8000/api/logs?filename=openmelon_error.log&lines=100"

# 列出所有日志文件
curl http://localhost:8000/api/logs/list
```

### 企业 Webhook 推送

将问答结果推送到钉钉 / 飞书 / 企业微信群聊：

```bash
# 推送到钉钉
curl -X POST "http://localhost:8000/api/webhook/dingtalk" \
  -H "Content-Type: application/json" \
  -d '{"question": "用户模块有哪些功能？", "answer": "用户模块包含登录、注册、密码重置..."}'
```

需要先在 `.env` 中配置对应平台的 Webhook URL。

### 思维导图

1. 切换到「思维导图」页面
2. 左侧编辑框输入 Markdown 标题层级：
   ```markdown

   ## 登录模块
   ### 用户名登录
   ### 验证码登录
   ## 注册模块

   ### 邮箱注册
   ```

3. 右侧实时渲染思维导图
4. 点击「查看导图」全屏查看

## 常见问题

### Q1：Neo4j 连接失败

页面提示数据库连接失败时，先刷新页面重试一次；若仍失败，请联系环境管理员检查服务状态。

### Q2：向量维度不匹配

错误：`Index query vector has 1024 dimensions, but indexed vectors have 1536`

这是因为切换了 LLM Provider（不同 Provider 的 Embedding 维度不同）。**解决**：重启后端服务（自动重建索引），然后重新上传文件。
这是因为历史数据和当前模型维度不一致。演示场景下建议：切换回原模型，或重新上传一份演示文档后再问答。

### Q3：上传文件卡住

先检查文件是否过大或格式是否受支持；可改用较小的 PDF/Markdown 文件重新上传。

### Q4：向量库不可用

演示不受影响，可以继续使用问答和图谱功能；“存入向量库”功能可暂时跳过。

### Q5：存入向量库失败

通常是当前环境未启用对应向量能力或临时不可用。建议先完成“生成+导出”演示，向量入库能力放到运维环境恢复后再验证。

## 总结

OpenMelon 提供了完整的文档智能问答和测试用例生成解决方案：

```
1. 上传文档   → 拖拽上传，后台异步解析 + 分块 + 构建知识图谱
2. 智能问答   → 意图识别 + 多通道检索 + LLM 生成（支持多步推理）
3. 探索图谱   → vis.js 可视化，搜索、筛选、点击查看详情
4. 生成用例   → AutoGen 三阶段智能体流水线 + 图谱知识驱动
5. 存入向量库  → 一键存储，支持语义检索已有用例
6. 覆盖率分析  → 指标卡片 + 排行图表 + 模块明细
7. 导入管理   → 文件追踪、重新索引、批量操作
```

详细的环境配置、API 参考和故障排查请查看 [MANUAL.md](MANUAL.md)。
